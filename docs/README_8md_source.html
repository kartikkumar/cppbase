6 `cppbase` is a template for a simple CMake-based C++ project: it can be used as the basis for new projects. An example CMake module is available to make it easy to include your project in other projects (should be customized): [Findcppbase.cmake](https://github.com/openastro/cmake-modules/blob/master/Modules/Findcppbase.cmake).
7 
8 (**N.B. remember to change all instances of `cppbase` to your new project name!**).
9 
10 Features
11 ------
12 
13  - General directory structure common to C++ projects
14  - Example of CMake module [Findcppbase.cmake](https://github.com/openastro/cmake-modules/blob/master/Modules/Findcppbase.cmake)
15  - Unit testing framework ([Catch2](https://www.github.com/catchorg/Catch2 "Catch2 Github repository") or [GTest](https://github.com/google/googletest "Google Test Github repository"))
16  - Install script (`make install`)
17  - CPack script for packaging (`make package`)
18  - Automatic API documentation ([Doxygen](http://www.doxygen.org "Doxygen homepage"))
19  - Continuous Integration ([Travis CI](https://travis-ci.org/ "Travis CI homepage"))
20  - Code coverage analysis ([Coveralls](https://coveralls.io "Coveralls.io homepage")) (`make coverage`)
21  - Example of how to include external dependencies (using `ExternalProject` module)
22  - Separate file to specify location of project files (`ProjectFiles.cmake`)

24 Requirements
25 ------
26 
27 To install this project, please ensure that you have installed the following (install guides are provided on the respective websites):
28 
29  - [Git](http://git-scm.com)
30  - A C++ compiler, e.g., [GCC](https://gcc.gnu.org/), [clang](http://clang.llvm.org/), [MinGW](http://www.mingw.org/)
31  - [CMake](http://www.cmake.org "CMake homepage")
32  - [Doxygen](http://www.doxygen.org "Doxygen homepage") (optional)
33  - [Gcov](https://gcc.gnu.org/onlinedocs/gcc/Gcov.html) (optional)
34  - [LCOV](http://ltp.sourceforge.net/coverage/lcov.php) (optional)
35 
36 The following dependencies are optional (see `Build options`):
37 
38  - [Catch2](https://www.github.com/catchorg/Catch2) (unit testing library necessary for `BUILD_TESTS` build option, with `TEST_FRAMEWORK=Catch2` (default))
39  - [GTest](https://github.com/google/googletest) (alternate unit testing library necessary for `BUILD_TESTS` build option and `TEST_FRAMEWORK=GTest`)
40 
41 These dependencies will be downloaded and configured automagically, if not already present locally (requires an internet connection).

43 Installation
44 ------
45 
46 Run the following commands to download, build, and install this project. Substitute `project_name` with the name of your project (if you leave it out, the repository will be cloned to a local folder called `cppbase`). This will customize the project targets for you. Note that `project_name` must not contain spaces! The ` --depth 1` parameter passed to `git clone` ensures that the git history is not downloaded. In case you would like to preserve the history of this project, omit that option.
47 
48  git clone https://www.github.com/kartikkumar/cppbase --depth 1 <project_name>
49  cd <project_name>
50  git submodule init && git submodule update
51  mkdir build && cd build
52  cmake -DPROJECT_NAME=<project_name> .. && cmake --build .
53 
54 To push this project to your own remote repository, you can run the following command, which will overcome the issues with utilizing a shallow clone:
55 
56  git commit --amend .
57 
58 This rewrites the last commit and ensures that you can then push the repository to a remote (e.g., Github, BitBucket, Gitlab, etc.).
59 
60 To install the header files, run the following from within the `build` directory:
61 
62  make install
63 
64 Note that dependencies are installed by fetching them online, in case they cannot be detected on your local system. If the build process fails, check the error log given. Typically, building fails due to timeout. Simply run the `cmake --build .` command once more.

66 Build options
67 -------------
68 
69 You can pass the following, general command-line options when running CMake:
70 
71  - `-DPROJECT_SUMMARY`: set short string summary for your project
72  - `-DPROJECT_VENDOR_NAME`: set short string name for vendor of your project
73  - `-DPROJECT_VENDOR_CONTACT`: set short string email address for vendor of your project
74  - `-DCMAKE_INSTALL_PREFIX[=$install_dir]`: set path prefix for install script (`make install`); if not set, defaults to usual locations
75  - `-DBUILD_SHARED_LIBS=[ON|OFF (default)]`: build shared libraries instead of static
76  - `-DBUILD_MAIN[=ON|OFF (default)]`: build the main-function
77  - `-DBUILD_DOXYGEN_DOCS[=ON|OFF (default)]`: build the [Doxygen](http://www.doxygen.org "Doxygen homepage") documentation ([LaTeX](http://www.latex-project.org/) must be installed with `amsmath` package)
78  - `-DBUILD_TESTS[=ON|OFF (default)]`: build tests (execute tests from build-directory using `ctest -V`)
79  - `-DBUILD_DEPENDENCIES[=ON|OFF (default)]`: force local build of dependencies, instead of first searching system-wide using `find_package()`
80 
81 The following commands are conditional and are only relevant if `BUILD_TESTS = ON`:
82 
83  - `-DBUILD_COVERAGE_ANALYSIS[=ON|OFF (default)]`: build code coverage using [Gcov](https://gcc.gnu.org/onlinedocs/gcc/Gcov.html) and [LCOV](http://ltp.sourceforge.net/coverage/lcov.php) (both must be installed; requires [GCC](https://gcc.gnu.org/) compiler; execute coverage analysis from build-directory using `make coverage`)
84  - `-DTEST_FRAMEWORK=[=GTest|Catch2 (default)]`: set unit testting framework for your project
85 
86 Pass these options either directly to the `cmake ..` command, e.g., to build the tests:
87 
88  cmake -DPROJECT_NAME=<project_name> -DBUILD_TESTS=on ..
89 
90 N.B.: Toggling options to build tests using `ccmake` does not work correctly, as the necessarily libraries are not downloaded automagically!

92 Project structure
93 -------------
94 
95 This project has been set up with a specific file/folder structure in mind. The following describes some important features of this setup:
96 
97  - `cmake/Modules` : Contains `CMake` modules
98  - `docs`: Contains code documentation generated by [Doxygen](http://www.doxygen.org "Doxygen homepage")
99  - `include`: Project header files (*.hpp)
100  - `scripts`: Shell scripts used in [Travis CI](https://travis-ci.org/ "Travis CI homepage") build
101  - `src`: Project source files (*.cpp), including `main.cpp`, which contains example main-function for project build
102  - `test`: Project test source files (*.cpp) that are provided to the selecting unit testing framework
103  - `.travis.yml`: Configuration file for [Travis CI](https://travis-ci.org/ "Travis CI homepage") build, including static analysis using [Coverity Scan](https://scan.coverity.com/ "Coverity Scan homepage") and code coverage using [Coveralls](https://coveralls.io "Coveralls.io homepage")
104  - `CMakeLists.txt`: main `CMakelists.txt` file for project (should not need to be modified for basic build)
105  - `Dependencies.cmake`: list of dependencies and automated build, triggered if dependency cannot be found locally
106  - `Doxyfile.in`: [Doxygen](http://www.doxygen.org "Doxygen homepage") configuration file, adapted for generic use within project build (should not need to be modified)
107  - `LICENSE.md`: license file for project (copyright statement needs to be edited)
108  - `ProjectFiles.cmake`: list of project source files to build
109  - `README.md`: project readme file, parsed as main page for [Doxygen](http://www.doxygen.org "Doxygen homepage") documentation
